# פרסום מוצרי 3D דרך GitHub

## הרעיון

כל פרויקט ב-Pro3duct יכול להישאר מקומי, אבל אפשר גם לפרסם אותו ל-GitHub כך שלכל מוצר יהיה אתר קטן וקישור עצמאי למודל שלו.

דוגמה:

- פרויקט מקומי: `X`
- ריפו GitHub: `pro3duct-x`
- אתר GitHub Pages: `https://shai7net.github.io/pro3duct-x/`
- קבצים באתר: `index.html`, `model.glb`, `twin-spec.json`, תמונות ותיאור מוצר

## מצב מומלץ ל-MVP

בשלב הראשון כדאי לתמוך בשתי אפשרויות:

1. `Local only` - הכול נשמר במחשב, בלי GitHub.
2. `GitHub Pages` - המשתמש מזין GitHub token, והמערכת יוצרת או מעדכנת ריפו לפרויקט.

## מה צריך כדי לממש

בצד backend:

- טבלת הגדרות פרסום לפרויקט: מצב פרסום, שם ריפו, branch, כתובת Pages.
- שירות GitHub שמקבל token מוצפן, יוצר ריפו, מעלה/מעדכן קבצים ומפעיל Pages.
- יצירת package סטטי לכל twin:
  - `index.html`
  - `model.glb`
  - `twin-spec.json`
  - `README.md`
  - thumbnails בעתיד

בצד frontend:

- כפתור `פרסום ל-GitHub` בעורך.
- שדה לשם הריפו או בחירה ביצירה אוטומטית.
- הצגת קישור סופי לכל מוצר.

## הערות חשובות

- הפרסום ל-GitHub צריך להיות אופציונלי בלבד.
- מפתח GitHub חייב להישמר מוצפן כמו מפתחות AI.
- עדיף להתחיל מ-repo נפרד לכל מוצר, כי זה נותן כתובת נקייה וברורה.
- בהמשך אפשר להוסיף גם מצב portfolio אחד שמכיל את כל המוצרים תחת כתובות שונות.
